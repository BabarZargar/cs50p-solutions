## 📚 Course Structure

### Week 0 — Functions, Variables
> *Where it all begins.*

The foundation. Learning how Python thinks — functions, arguments, return values, and the beauty of `print()`. The moment "hello world" runs for the first time and you realize you can make a computer do whatever you want.

**Key concepts:** `input()`, `print()`, string formatting, f-strings, `int()`, `float()`, `round()`

---

### Week 1 — Conditionals
> *Teaching the computer to make decisions.*

If this, do that. Else, do this. The logic that powers everything from traffic lights to stock markets — and now, your Python programs.

**Key concepts:** `if`, `elif`, `else`, comparison operators, boolean expressions, `match`

---

### Week 2 — Loops
> *Why do something once when you can do it a thousand times?*

Loops unlock the real power of programming. Repetition, iteration, and the art of knowing when to `break`.

**Key concepts:** `while`, `for`, `break`, `continue`, lists, `range()`

---

### Week 3 — Exceptions
> *Because things go wrong. A lot.*

The real world is messy. Users type wrong things. Files don't exist. Servers go down. Exceptions teach you to expect the unexpected and handle it gracefully.

**Key concepts:** `try`, `except`, `else`, `raise`, `ValueError`, `KeyError`, defensive programming

---

### Week 4 — Libraries
> *Don't reinvent the wheel.*

Why write code that already exists? Libraries let you stand on the shoulders of giants. Also where the real fun begins.

**Key concepts:** `import`, `from`, `random`, `statistics`, `sys`, `pip`, third-party packages

---

### Week 5 — Unit Tests
> *How do you know your code works? Test it.*

Writing code is only half the job. The other half is proving it works. Unit testing changed how I think about writing functions forever.

**Key concepts:** `pytest`, `assert`, test functions, edge cases, `raise`, test-driven thinking

---

### Week 6 — File I/O
> *Making your program remember things.*

Programs that forget everything when they close are useless. File I/O makes data persist — reading CSVs, writing text files, processing images.

**Key concepts:** `open()`, `read()`, `write()`, `csv`, `PIL`, `with`, file modes

---

### Week 7 — Regular Expressions
> *The dark arts of pattern matching.*

Regex looks like someone fell asleep on a keyboard. Then you realize it can validate an email, extract a phone number, or parse an entire document in one line. Mind-blowing.

**Key concepts:** `re`, `re.search()`, `re.match()`, `re.sub()`, patterns, groups, `re.IGNORECASE`

---

### Week 8 — Object Oriented Programming
> *Everything is an object. Everything.*

OOP changes how you see code — not as a list of instructions but as a collection of objects that interact with each other. Classes, instances, methods, properties.

**Key concepts:** `class`, `__init__`, `self`, instance methods, `@property`, `@classmethod`, inheritance, operator overloading

---

### Week 9 — Et Cetera
> *The final boss. The stuff that makes your code actually Pythonic.*

The cherry on top. The things that separate Python code that works from Python code that's beautiful.

**Key concepts:** type hints, docstrings, `*args`, `**kwargs`, list comprehensions, dictionary comprehensions, generators, `yield`

## 🗃️ Final Project — Stash

> *"We constantly save things to remember later. We never find them again."*

While this repo contains my problem set solutions, my final project lives in its own repository because it deserves the space.

**Stash** is a CLI tool that acts as your second brain. Dump anything into it — a link, a meeting time, a password, a random idea — and it automatically categorizes it using regex pattern matching and keyword scoring. Retrieve anything instantly by keyword or category.

Built with: Python · Regex · JSON · pytest · pytz
